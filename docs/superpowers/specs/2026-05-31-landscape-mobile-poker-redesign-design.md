# 横屏移动端牌桌界面重设计

**日期**：2026-05-31
**范围**：仅横屏移动端（`@media (max-width: 940px) and (orientation: landscape)`）
**目标**：在有限的横屏空间里，让牌桌尽可能大、操作贴近手牌、信息按需展开。

---

## 1. 现状问题

横屏下（约 940px 宽 × 较矮高度），现有布局把空间切成主桌列 + 220px 右侧栏，再上下塞「我的手牌」「轮到你行动」两个独立卡片，结果：

- 牌桌被压扁，几乎无法看清。
- 操作面板占用大量纵向空间，按钮与手牌分离。
- 玩家座位卡片信息冗余（「在线」「已补」「active」全都文字罗列）。
- 加注子面板含 4 个底池比例快捷键 + 输入框 + 下注/加注两个按钮，水平/垂直都很挤。

---

## 2. 设计原则

1. **桌面最大化**：牌桌占据屏幕主区，椭圆（`border-radius: 50%`），其他玩家围桌排布。
2. **就近操作**：操作按钮贴在手牌右侧，按需逐级展开。
3. **信息按需展开**：座位卡片极简，详情通过 Modal 查看；右侧栏整体取消，所有信息收敛到顶条入口。

---

## 3. 整体布局

```
┌──────────────────────────────────────────────────────────┐
│ 房间#XXXX  盲注 5/10·Straddle·2/9  [轮到你 38s] [房主设置][💬][我][退出] │
│ ┌──────────────────────────────────────────────────────┐ │
│ │     [玩家2]   [玩家1]   [玩家3]                          │ │
│ │  [玩家8]    ◯ 公共牌+底池        [玩家4]                 │ │
│ │     [玩家7]               [玩家5]                        │ │
│ │              ♠♣ 我的手牌  [弃牌]                         │ │
│ │                          [跟注X]                         │ │
│ │                          [加注]                          │ │
│ └──────────────────────────────────────────────────────┘ │
│              [我·昵称 筹码 ●]                              │
└──────────────────────────────────────────────────────────┘
```

- **顶条**：固定高度约 36–40px 的横条，所有"非牌局"功能入口集中在右侧。
- **主体**：椭圆牌桌，占满主列，纵向尽可能高（约 `100dvh - 顶条 - 自己锚点`）。
- **自己锚点**：贴在椭圆桌正下方中央外侧，单行：昵称 + 筹码 + 在线绿点。
- **右侧栏**：横屏下整体取消。所有右栏内容拆分到顶条入口。

---

## 4. 顶条

横屏下顶条结构（从左到右）：

| 区块 | 内容 |
|------|------|
| 房间信息 | `房间 #XXXX` + 「盲注 5/10 · Straddle · 2/9」 |
| 行动提示 | `[轮到你 38s]` 胶囊（仅 `isMyTurn` 时显示，金色背景） |
| 房主设置 | 按钮（仅房主可见），点击切换现有 `host-settings-popover` |
| 💬 消息 | 按钮，所有人可见，点击打开「补码申请抽屉」；有 pending 申请时按钮上加红点 |
| 我 | 圆形头像样式按钮（昵称首字 + 在线绿点角标），点击打开「我的资料 Modal」 |
| 退出 | 直接退房 |

**轮到自己** 的金色横幅由原中央独立卡片改为顶条胶囊，不再占用纵向空间。

---

## 5. 牌桌

- 椭圆形：`border-radius: 50%`（不是胶囊），保留绿色台面 + 木框。
- 占满主列宽，高度 = 顶条以下、自己锚点以上。
- 公共牌 + 底池居中。
- `actorId` 属于其他玩家时，仍在中央显示「XXX 决策中 · 38s」（保留）。

### 5.1 座位（其他玩家）

- 1–8 号座位均匀分布在椭圆**上半圈 + 左右两侧**，**避开底部弧线**（留给自己）。
- 卡片瘦身，**移除头像**，**仅显示**：
  - 昵称（一行，超出省略）
  - 筹码量
  - **在线状态绿点**（在线绿、离线灰）
- `acting`：金色描边 + 倒计时小药丸（保留）。
- 卡片宽度从 108px 压到约 96–100px。
- **整张卡片可点击** → 打开「玩家详情 Modal」。

### 5.2 自己

- **不进入椭圆桌的座位序列**。
- 锚点元素紧贴椭圆桌底部正下方外侧，单行极简：昵称 + 筹码 + 在线小绿点。
- `acting` 时锚点也加金色描边。
- 自己**不可点**自己的锚点查看详情（自己的信息已在顶条「我」入口）。

### 5.3 手牌

- 不再用独立的「我的手牌」卡片。
- 手牌横向并排，浮在椭圆桌**底部内沿**（即自己锚点正上方、桌内）。
- 牌大小延续现有横屏尺寸（28×38 左右）。

---

## 6. 操作按钮（轮到自己时）

**位置**：手牌**右侧**，纵向从上到下紧凑排列。

**可见按钮规则**：
- **弃牌**：始终显示（红色调）。
- **跟注 X / 过牌**：二选一显示，`canCall` 显示「跟注 X」，`canCheck` 显示「过牌」，否则隐藏整个槽位。
- **加注 / 下注**：`canRaise` 显示「加注」，`canBet` 显示「下注」，否则隐藏整个槽位。

按钮尺寸：约 64×34 px，竖向堆叠，间距 4–6px。

### 6.1 Straddle 决策

`game.street === 'straddleDecision'` 且自己是 actor 时，操作按钮列替换为两个按钮：「Straddle X」「不 Straddle」（不显示弃牌/跟注/加注）。

---

## 7. 加注子面板

**触发**：点「加注」或「下注」按钮 → 在按钮列右侧横向展开。再次点同按钮 / 点其他按钮 / 行动结束 → 关闭。

**布局**（横向一行）：
```
[ − ] [ 输入框 ] [ + ]  [All-in]  [确认 加注到 X]
```

| 元素 | 行为 |
|------|------|
| 输入框 | 默认值 = `legalActions.minAmount`；可直接输入；范围 `[minAmount, maxAmount]` |
| − / + | 步长 = `room.config.bigBlind`；clamp 到合法区间；越界时按钮 disabled |
| All-in | 一键将输入框值设为 `legalActions.maxAmount`（不立即提交，仍需点确认） |
| 确认 | `canRaise` 显「加注到 X」，`canBet` 显「下注 X」；值不合法时 disabled |

成功 act 后清空 `betAmount` 并关闭子面板。

**移除**：现有的 `底池 1/3 / 1/2 / 2/3 / 1` 四个比例快捷键，全部去掉。

---

## 8. 弹层与抽屉

| 名称 | 触发 | 形式 | 内容 |
|------|------|------|------|
| `PlayerDetailModal` | 点其他玩家座位卡片 | 居中 Modal + 灰色蒙层 | 昵称（带房主标记）/ 在线状态 / 当前筹码 / 累计补码 / 当前局状态（active/folded/all-in/等待）/ 是否本局行动中 |
| `MyProfileModal` | 点顶条「我」 | 居中 Modal | 昵称编辑 + 更新 / 我的筹码 / 累计补码 / 申请补码输入 + 提交 |
| `TopUpRequestsDrawer` | 点顶条「💬」 | 右侧侧滑抽屉 | **仅** pending 补码申请列表；房主可见「通过/拒绝」按钮；批准仅在 `canRequestTopUp` 状态下生效（非游戏中），批准后服务端真正加筹码并通知申请人 |
| `SettlementModal` | 房主点设置里的「结算」 | 居中 Modal | 所有人输赢总表（同现有 `settlement` 数据），手动关闭 |
| `HostSettingsPopover` | 房主点顶条「房主设置」 | 现有 popover | 开始下一局 / 设置密码 / 结算 / 解散房间（不变） |

### 8.1 通知（非补码申请）

原右侧栏 `notifications` 列表里的非补码消息（玩家加入/退出、发牌提醒等）改为**顶部 toast**，3 秒后自动消失，不进抽屉。

### 8.2 关闭交互

- Modal：点蒙层 / 点 X 关闭。
- 抽屉：点蒙层 / 滑动关闭 / 点 X 关闭。
- popover：再点触发按钮 / 点外部关闭。

---

## 9. 不变的部分

- 服务端 socket 协议、`LegalActions` 数据结构、牌局核心引擎不动。
- 桌面端（≥940px）布局不动。
- 竖屏移动端（≤640px）布局不动。
- `App.tsx` 顶层状态、socket 监听逻辑不动。

---

## 10. 实现范围

修改文件：
- `src/App.tsx`：新增 4 个 Modal/Drawer 子组件（同文件），重构 room-mode 渲染结构（顶条 + 牌桌 + 自己锚点 + 操作按钮列 + 加注子面板）。
- `src/styles.css`：在 `@media (max-width: 940px) and (orientation: landscape)` 块内重写关键样式；在通用规则中新增 Modal/Drawer 样式。
- 不引入新依赖。
- 不新增独立组件文件。

---

## 11. 验收

- 横屏 940×420 之类的典型移动设备视口下：
  - 牌桌椭圆可清晰显示公共牌、底池、其他玩家。
  - 自己手牌、操作按钮列贴近自己锚点。
  - 顶条不溢出。
  - 「我」「💬」入口可点开对应弹层；「💬」红点正确反映 pending 补码申请。
  - 加注子面板 +/- 步长为 1 BB，All-in 填最大值。
  - 不破坏桌面端与竖屏端展示。
